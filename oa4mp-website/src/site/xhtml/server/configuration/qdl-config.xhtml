Configuring QDL.

There is really nothing to say here, except to point to the documentation on the main website, and to list the allowed additional attributes in the qdl tag

Name Default Description
strict_acls false Enabled/disable enforcing strict access control for scripts. If this is set true, then every script must have permission granted to execute.

Naming

If there is a single configuration, that may be named anything you like and it will be used. If there are multiple configurations, then only the one named qdl-default will be loaded. This allows you to have several for testing and just activate the one you need.
